As a style guide, we usually write inner classes at the end of the enclosing class—after the methods.


Nonstatic member class

Ponasa se kao i svaki 2. instance member outer klase. Ne moze se kreirati bez instance outer klase, zato se i naziva nonstatic jer zahteva instancu outer klase. Posto je outer objekat obavezan, inner objekat ima hidden referencu ka outer objektu.

Koristimo ih kada objekat inner klase zahteva pristup objektu outer klase. Nonstatic member klase cesto imaju ulogu adaptera, tj. kada zelimo da istu klasu predstavimo na razlicite nacine, npr.:

HashMap-a implementira Map interface. Kada pozovemo HashMap#entrySet() dobicemo instancu EntrySet inner klase koja predstavlja Set sa Map.Entry-ma. Kada pozovemo EntrySet#iterator() dobicemo objekat interne klase EntryIterator koja pak ima metod next() koji vraca objekat Node inner klase.

Outer object nece biti GC-ovan ako postoji inner objekat jer taj inner objekat ima hidden referencu ka outer objektu.

Obuka Udemy, Java in depth, 262. ep.


Anonymous klase

Deklarisemo ih i instanciramo na istom mestu, tamo gde je to potrebno.

Koristimo ih da bi izrazili odredjenu strategiju. Npr., pri kreiranju TreeSet-a mozemo koristiti anonymous klasu kako bi definisali strategiju koja ce se koristiti za sortiranje elemenata Set-a. Compiler generates a new class file for each anonymous inner class. The filename usually looks like ClassName$1, where ClassName is the name of the class in which the anonymous inner class is defined, followed by a dollar sign and a number. The generation of many class files is undesirable because each class file needs to be loaded and verified before being used, which impacts the startup performance of the application. Takodje, instanca anonymous klase se kreira svaki put kada se pozove kod gde je anonymous klasa definisana, sto moze biti skupa operacija (because of this we often use anonymous classes where only a single instance of the implementation will be required). Da bi se ovo sprecilo mozemo koristiti:

Obrati paznju da instanciramo interface i zato je ovo anonymous klasa koja ce se samo 1 kreirati i onda je mozemo koristiti na vise mesta. Posto je ovo u sustini izraz, moramo staviti ; iza body-a anonymous klase. Sada STR_LENGTH_COMPARATOR mozemo proslediti svuda gde se zahteva Comparator instanca. Medjutim ako zelimo da prosledimo 2 string-a compare() metodu u okviru ove anonymous klase, onda to mozemo uraditi sa:

STR_LENGTH_COMPARATOR.compare("java", "javascript");

Ovo je isto kao da smo kreirali klasu koja implementira Comparator interface i u njoj definisali compare metod. Ovde isto kreiramo novu klasu ali joj nigde ne definisemo ime, koristimo samo interface i zato kompajler koristi sufiks u obliku $1. Ovo ne moramo raditi samo sa interface-om, mozemo koristiti i apstraktnu klasu ili regularnu klasu ako zelimo da kreiramo novu klasu koja ih extend-uje. Ali posto nigde ne definisemo ime te nove klase onda je ona anonimana klasa.

Kada se anonymous klasa koristi unutar metoda onda ona ima pristup i lokalnim promenljivama i parametrima metoda, medjutim, ovo vazi samo ako su te lokalne promenljive i parametri deklarisani kao final.

Anonymous inner classes are a useful construct for implementing event listeners.

Obuka Udemy, Java in depth, 263. ep.


Static member classes

Koristimo kada nam treba nested klasa kojoj ne treba pristup member-ima outer klase. Static member class se naziva static zato sto je ona zapravo staticki member outer klase. Posto je ona staticki member to znaci da joj mozemo pristupiti bez da postoji instanca outer klase, isto kao sto mozemo pristupiti statickim metodama i poljima klase bez da postoji instanca te klase. Instanciranje static member klase radimo na sledeci nacin:

Primer 1. use case-a je Node static member klasa u okviru HashMap-e. HashMap-a koristi instance Node klase kako bi u njima cuvao key-value parove ali, sa 2. strane, Node objektu ne treba pristup podacima iz outer klase (static member klasa nema hidden referencu ka outer objektu sto znaci da zauzima manje prostora od nonstatic member klase).


A static member class has access to all static members of its containing class, including private members. The reverse is true as well: the methods of the containing class have access to all members of a static member class, including the private members. A static member class even has access to all the members of any other static member classes, including the private members of those classes.


A static member class cannot have the same name as any of its enclosing classes. In addition, static member classes and interfaces can be defined only within top-level classes and in other static member classes and interfaces. This is actually part of a larger prohibition against static members of any sort within member, local, and anonymous classes.

Obuka Udemy, Java in depth, 265. ep.
